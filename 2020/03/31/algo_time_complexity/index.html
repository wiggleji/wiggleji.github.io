시간복잡도 (Time Complexity)

알고리즘 공부에서 절대 빠질 수 없는 요소가 바로 시간복잡도이다. 많은 코딩테스트에서도 따지는 부분이며, 실제 업무에도 많이 활용되기 때문에 시간복잡도는 최적화 공부에 도움이 되는 부분이다.

인프런에서 무료로 강의를 제공하고 있다.
-> 링크

시간 복잡도(Time complexity)는 코드의 실행 시간을 측정하여 알고리즘의 효율을 따지는데 필요하다.

아래 예제 코드를 보자.

print_hello 함수는 인자로 입력받은 number만큼 Hello를 출력한다. 위 함수의 실행 횟수를 주석으로 적어보자.

함수 실행시, 맨 처음 출력문은 한번만 실행되지만,
Hello!를 출력하는 부분은 for loop문으로 묶여 있기 때문에 인자값 number에 따라 N번 횟수만큼 실행이된다.

위의 실행시간를 수식으로 1+N 처럼 나타낼 수 있다.

이러한 성능 분석을 수식처럼 표현하기 위해 사용되는 표기법을 점근 표기법이라고 하는데, O(빅오), Ω(오메가), ⍬(세타) 이렇게 3가지 방식이 있다. 알고리즘의 성능에서 여러 케이스를 나누어 측정하는데, best, average, worst 이렇게 3가지 케이스를 가지고 알고리즘의 성능을 판단한다.

보통 O(빅오) 표기법을 많이 사용하며, 빅오 표기법은 최악의 경우로 계산한다. 빅오 표기법에서 가장 흔히 사용되는 수식은 아래와 같다.

Big-O Notation

  • O(1) : 상수형 (constant)
  • O(log N) : 로그형 (log)
  • O(N) : 선형 (linear)
  • O(N^2) : 제곱형 (quadratic)
  • O(2^N) : 거듭제곱형 (power)
  • O(N!) : 계승형 (factorial)

상수형이 가장 빠르며, 지수형에 가까워질수록 시간이 오래걸린다. 이를 통해, 작성한 코드의 효율을 확인할 수 있다.
성능순서: O(1) < O(logN) < O(NlogN) < O(N^2) < O(2^N) < O(N!)

1. O(1) 상수형 (constant)

상수형은 오직 한 단계로만 실행된다.
데이터 양이 적든 많든 상관 없이 일정한 실행 시간이 필요하다.

2. O(log N) 로그형 (log)

logN 그래프를 보면 알겠지만, 데이터양이 많아져도 실행시간이 선형적으로 증가하는 것보다 훨씬 천천히 증가하고, 시간에 비례하다. 선형보다 데이터양이 2^N배 많으며, **이진탐색(Binary Search)**이 로그형 시간복잡도를 가진다.

조금의 시간이 더 소요되지만,
정렬 알고리즘에서 퀵정렬(Quick Sort), **합병정렬(Merge Sort)**이 O(N log N) 시간복잡도르 가진다. 정렬을 할 때, 모든 데이터를 작은 단위로 쪼개어 비교하며 조회하기 때문에, N배 더 많은 시간이 소모된다.

3. O(N) 선형 (linear)

가장 보편적인 시간복잡도로, 데이터양과 시간이 정비례하다. 단일 for문과 같이 데이터 전체를 탐색하는 경우, 선형 복잡도를 가진다.

4. O(N^2) 제곱형 (quadratic)

선형의 제곱만큼의 시간이 걸리며, 보통 2중 for문을 사용할 때 나타나는 시간복잡도이다. N값에 따라 시간이 매우 길어지며, 효율이 매우 떨어진다. 정렬 알고리즘에서는 삽입정렬(Insertion sort), **버블정렬(Bubble sort)**가 있다.

5.O(2^N) 거듭제곱형 (power), O(N!) : 계승형 (factorial)

제곱형부터 이미 시간복잡도가 급격하게 증가하기 때문에, 거듭제곱형과 계승형은 시간복잡도를 따지기에는 너무 큰 기울기를 가진다. 게승형은 심지어 화면에 표현할 수도 없을 정도로 증가하기 때문에.. 여기까지만 알아보자
